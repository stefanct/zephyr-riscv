Printing policy a) logging (SYS_LOG_<>) stuff that should vanish in time-critical builds -> can be thrown out of code by Kconfig/logging b) (verbsoity controlled) printkv stuff in tests that is not time critical -> can control via print_set_verbosity -> stays in binary, even when vebosity set to not show -> productive code should avoid compiling test code anyway c) buffered LOG_PERF stuff inside time-critical code that needs to be printed for debug -> logs to buffer and prints after run -> still slower than not printing, better than serial out -> set CONFIG_APP_LOG_PERF_BUFFER_DEPTH=0 to deactivate
